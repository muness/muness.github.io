# Aim: Executable Expectations Series

## Desired reader outcome

Readers should understand why LLM-enabled domain systems need more than better tickets, summaries, prompts, or code generation. They should see how local expert correction can become durable system learning when it is captured as an executable expectation with provenance, scope, review, and regression evidence.

## Series intent

Place the executable-expectations work beside the 2026-06-13 agent-domain-runtime essays:

- `Agents Don't Learn the Domain. The System Does.` establishes the domain harness.
- `There Is No Agent Workflow Runtime.` establishes the workflow/runtime boundary.
- `Agent Work That Changes Behavior.` establishes behavior change as the outcome.
- The new series should explain organizational communication, communication reversal, process reliability, and the concrete Recon Agent harness.

## Candidate essay roles

1. **Organizational Communication**
   - Job: explain communication as the medium of organizational memory, coordination, authority, and control.
   - Claim: organizations scale by turning local judgment and action into communicable artifacts, records, roles, routines, and decisions.
   - Output: the reader understands what communication did before reversal becomes the problem.

2. **Organizational Communication Reversal**
   - Job: map McLuhan/Mir media reversal to organizational communication.
   - Claim: communication pushed to abundance reverses into ceremony, action-substitution, AI-accelerated mutation, and amnesia.
   - Output: executable expectations become a counter-medium, not just better documentation.

3. **Variable and Standardized Process Reliability**
   - Job: use Potter's frame to explain how the archaeological record becomes specific and reliable.
   - Claim: domain work needs variable entry points for judgment and standardized process/components/systems for safe promotion.
   - Output: a bridge between conceptual reversal and the concrete Recon Agent harness.

4. **Recon Agent Harness**
   - Job: show the actual mechanism without naming the client.
   - Claim: a position reconciliation failure can move from SME correction to schema'd spec, code/prompt/test change, deterministic regression, and named regression case.
   - Output: make Dave's `too abstract` objection impossible by walking through the built workflow.
