# Open Horizons Project Memory

This tree keeps project-local strategic memory for the Muness blog: aims, outcomes, metis, guardrails, and session notes that should travel with the writing repo.

It is intentionally repo-native. Blog essays are not just posts; they are evidence of evolving claims, mechanisms, and writing constraints. Keeping the alignment artifacts beside the posts makes later edits less dependent on conversational memory.

## Structure

- `aims/` — outcomes the writing should create for readers and for the body of work.
- `outcomes/` — observable signals that an aim is working.
- `metis/` — reusable lessons discovered while drafting, reviewing, or publishing.
- `guardrails/` — constraints that should prevent repeated writing failures.
- `sources/` — source excerpts, quote ledgers, argument spines, and charts that should travel into future drafts.
- `sessions/` — dated working notes and problem framings.
