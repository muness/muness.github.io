# I help teams change how they work with agents.

A client brings me a workflow that is slow, brittle, expensive, or trapped in a few people's heads. The first question is often, “Where should we use an agent?”

I start with the people, steps, constraints, and owner. We decide where model judgment helps, where ordinary code is safer, and what evidence will tell us the change worked. Then we build one path through the workflow and test it on cases the team knows.

The goal is a change in behavior. Someone should be able to make a better decision, preserve an expert correction, or stop rebuilding context every time the work starts.

## Client work

I keep the organizations unnamed. Recent systems include:

- **An innovation connector** over about 200,000 expert profiles, with a trace showing why each match ranked. [Read the technical case.](/posts/hybrid-search-case-study-ranking-better-results/)
- **Data pipelines shaped by the people who knew the business rules**, without asking them to write code. Distinguishing a code bug from a missing rule led to [Sketch-CE](https://github.com/open-horizon-labs/counterexample-supplemented-sketches).
- **Performance reviews and career-growth plans** drafted from 360-degree feedback across projects, an internal skills rubric, and past reviews.
- **Project staffing suggestions** using project briefs, availability, prior assignments, expertise, and the same performance evidence.

Models helped search, synthesize, or generate. People made the decisions.
