# Preface {.unnumbered}

This novella is written as a lightly fictionalized "agency novel" in the tradition of Goldratt's *The Goal* (constraint-driven plot) and Lencioni's fables (named dysfunctions, actionable takeaways). Each episode is a client engagement that forces the team to make one Open Horizons move real.

The commands, vocabulary, and artifacts are real. The clients are composites. The lessons are hard-won.

---

# The Commands {.unnumbered}

Open Horizons uses nine commands organized into three phases: **Grounding**, **Execution**, and **Reflection**.

## Grounding --- Understand the Problem

**/problem-space**
: Map what we're choosing to optimize and what constraints we treat as real. The problem space is the terrain where solutions live.

**/problem-statement**
: Define or refine the framing. Change the statement, change the solution space.

**/aim**
: Clarify the outcome you want---a change in user behavior, not a feature shipped. Includes mechanism (the causal lever) and feedback (the signal that validates).

## Execution --- Build & Ship

**/solution-space**
: Explore candidate implementations. Four levels: Tarp (patch symptom), Nearest Peak (optimize current framing), Beyond the Nearest Peak (reframe), Terraform (redesign terrain).

**/execute**
: Do the work. Pre-flight, build, detect drift, salvage if needed. Steer or Abort---never thrash.

**/ship**
: Deliver to users. Optimize the delivery path---the path from merged code to working install.

## Reflection --- Capture Learning

**/review**
: Check work and detect drift. A second opinion before committing.

**/dissent**
: Structured disagreement. "What would make this wrong?"

**/salvage**
: Extract learning before restarting. Code is a draft; learning is the asset.

# Vocabulary {.unnumbered}

**Aim**
: The outcome you want---a change in user behavior, not a feature shipped.

**Mechanism**
: The causal lever you believe will move the outcome. Your hypothesis about how the world works.

**Feedback**
: The signal that validates or disproves the mechanism.

**Guardrail**
: An explicit constraint with boundary, reason, and revisit trigger. What you won't do, and why.

**Metis**
: Practical wisdom. Tacit knowledge that compounds through experience and reflection.

**Dive Pack**
: Minimal pre-flight context: aim, constraints, landmines, and learnings from previous sessions.

**Manifest**
: "Defaults in a box" for a project: tools, session injections, build/review/ship norms.

**Terraform**
: Reimagine the landscape so the problem dissolves. Not patching, not optimizing---reshaping reality within physics constraints.

**Tarp**
: Add a knob, tweak a setting, patch the symptom. Rational under time pressure; toxic as a default.

**Nearest Peak**
: Optimize inside the current framing. Better, but still trapped by assumptions.

**Beyond the Nearest Peak**
: Step back to the problem statement and search for a different approach entirely.

**Delivery Path**
: The path from merged code to working install. When execution is cheap, delivery is the work.

**Delivery-Path Tax**
: The friction between code and usable value: review-to-merge time, gate/scan time, manual approvals.

**Chaos Dragon**
: Thrash at speed---agents accelerating the wrong things without grounding. The failure mode of ungoverned AI.

**Salvage Loop**
: Extract learning, encode in working memory, restart clean.
